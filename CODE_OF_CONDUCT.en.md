# Code of Conduct

[简体中文](CODE_OF_CONDUCT.md) | [English](CODE_OF_CONDUCT.en.md)

Nami Mail aims to be a project where users and contributors can collaborate as equals and with professionalism. When participating in issues, discussions, pull requests, code review, releases, and other project spaces, communicate with respect, clarity, and verifiable evidence.

## Expected Behavior

- Discuss code, design, evidence, and user impact instead of judging people.
- Respect differing experience, language, region, identity, and ways of working.
- Give specific, constructive feedback about security, privacy, accessibility, and reliability.
- Acknowledge uncertainty, provide reproducible evidence, and correct mistakes promptly.
- Respect the sensitivity of mail content and account data, and minimize disclosure by default.

## Unacceptable Behavior

- Harassment, discrimination, insults, threats, stalking, malicious humiliation, or personal attacks.
- Disclosing another person's identifying information, mail, attachments, credentials, tokens, logs, or screenshots without permission.
- Posting malicious code, social-engineering material, or attack instructions that can directly harm others in community channels.
- Repeatedly interrupting discussions, spamming, arguing in bad faith, or impeding collaboration with false statements.
- Circumventing the project's security disclosure process or retaliating against a reporter.

## Scope

This code applies to the repository, issues, pull requests, discussions, release notes, real-time communication, and any activity conducted under the Nami Mail project name.

## Reporting and Handling

Do not disclose sensitive mail data, credentials, tokens, or details that could be used to harm others in a public thread. When a conduct concern also involves a security vulnerability, privacy disclosure, account risk, or exploitable technical detail, use [GitHub private vulnerability reporting](https://github.com/QinIndexCode/nami-mail/security/advisories/new). This is the enabled private security-reporting route and must not be used for ordinary conduct disputes.

The project currently has no dedicated private Code of Conduct email address or private conduct-reporting channel. For a general conduct concern that contains no sensitive information, a public issue may include only the minimum necessary facts and relevant links. If private communication is needed, first create a contact request that contains no incident details and ask a maintainer to provide a verifiable private channel. Do not treat the security-reporting route, an invented email address, or contact information absent from the repository homepage as a general conduct-reporting channel.

Maintainers will make decisions based on available evidence, the safety of the people involved, and confidentiality needs. Actions can include clarification, a reminder, content removal, limiting participation, or a ban. Retaliation against a good-faith reporter is unacceptable.

## Enforcement Principles

Maintainers will aim for fairness, timeliness, and the minimum necessary disclosure. Content involving a security vulnerability must still follow the [Security Policy](SECURITY.en.md), rather than being handled publicly through the Code of Conduct process.
