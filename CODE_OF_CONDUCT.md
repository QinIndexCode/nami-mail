# 社区行为准则

Nami Mail 希望成为一个让用户和贡献者能够平等、专业地协作的项目。参与 Issue、讨论、Pull Request、代码审查、发布和其他项目空间时，请以尊重、清晰和可验证的方式交流。

## 期望的行为

- 就代码、设计、证据和用户影响讨论问题，而不是评价个人；
- 尊重不同经验、语言、地区、身份和工作方式；
- 对安全、隐私、可访问性和可靠性提出具体、建设性的反馈；
- 承认不确定性，给出可复现证据，并在发现错误后及时更正；
- 尊重邮件内容与账户数据的敏感性，默认最小化披露。

## 不可接受的行为

- 骚扰、歧视、侮辱、威胁、跟踪、恶意羞辱或人身攻击；
- 未经许可披露他人的身份信息、邮件、附件、凭据、令牌、日志或截图；
- 在社区渠道发布恶意代码、社会工程内容或可直接伤害他人的攻击说明；
- 反复打断讨论、刷屏、恶意争论，或以不实陈述阻碍协作；
- 规避项目的安全披露流程或对报告者实施报复。

## 适用范围

本准则适用于项目仓库、Issue、Pull Request、讨论、发布说明、即时沟通和任何以 Nami Mail 项目名义开展的活动。

## 报告与处理

请不要在公开线程举报行为问题，也不要在举报中附上无关的敏感邮件数据。请使用仓库主页中维护者公布的非公开渠道报告，并说明发生位置、时间、相关链接和你希望获得的处理方式。项目尚未配置专门行为准则邮箱，不会在此文档中虚构一个联系方式。

维护者会在可用证据、当事人安全和保密需要之间作出判断。处理措施可包括澄清、提醒、删除内容、限制参与或禁止参与。任何对善意举报者的报复都不可接受。

## 执行原则

维护者会尽量保持公正、及时和最小必要披露。涉及安全漏洞的内容仍应遵循 [安全策略](SECURITY.md)，而不是在行为准则流程中公开处理。
