# Nami Mail 0.1.1

[简体中文](v0.1.1.md) | [English](v0.1.1.en.md)

Nami Mail 0.1.1 新增中英文界面与可控的邮件翻译，并完善了账户添加、邮件状态和桌面更新反馈。

## 下载

请只下载本 Release 中的 `Nami Mail Setup 0.1.1.exe`。版本化 ZIP、JSON 清单、`latest.yml` 和 `.blockmap` 是发行与自动更新内部资源，不是手动安装包。

## 本次更新

- 可在简体中文与 English 间切换；系统通知、最小化到托盘确认和 OAuth 完成页会跟随所选语言。项目的界面语言包采用 JSON 文件维护，并提供扩展文档。
- 新增可选的邮件正文机器翻译：在设置中配置兼容 LibreTranslate 的服务后，可按需翻译当前打开的邮件。译文可选择和复制，并明确提示机器翻译可能不准确。
- 翻译完全由用户主动触发：只有点击翻译后，当前邮件的纯文本正文才会发送到所选服务；服务地址和可选 API Key 加密保存在本机。
- 改进账户添加和邮件状态：中文输入法输入不会被打断，自定义域名检测不会在输入过程中抢走焦点；归档、移至废纸篓和未读状态在与服务器核对期间保持一致，包含 Gmail All Mail 场景。
- 发送状态现在区分“已提交”和“已在发件箱中确认”；更新问题会分别说明网络、TLS、签名或完整性、发布资源与缓存清理状态，便于恢复操作。

## 签名与 SmartScreen

此版本安装程序未进行 Authenticode 签名。Windows SmartScreen 可能显示未知发布者或来源警告。请回到官方 Release 页面核对仓库、tag、安装程序名称和本说明；任何信息不明确时应停止安装，而不是绕过系统警告。

## 自动更新状态

本 Release 使用 Ed25519 签名的版本化 JSON 清单绑定更新 ZIP 的版本、名称、大小和 SHA-512。它是应用内更新信任材料，不是手动安装文件。

从 `0.1.0` 到 `0.1.1` 的公开 GitHub Release 真实升级验收尚未完成。本说明不把本地构建、CI、清单签名、ZIP 下载或 Release 资源存在本身表述为线上自动更新已验收。

## 已知限制

- 本版本仅提供 Windows x64 安装程序。
- 邮件翻译需要可用的 LibreTranslate 兼容服务；机器翻译仅供参考，所选邮件的纯文本正文会在用户主动翻译后发送给该服务。
- 部分邮箱服务商要求 OAuth、应用专用密码或客户端授权码，具体准备步骤见邮箱接入指南。

更多帮助见 [Windows 安装与更新指南](https://github.com/QinIndexCode/nami-mail/blob/main/docs/INSTALLING.md)、[邮箱接入指南](https://github.com/QinIndexCode/nami-mail/blob/main/docs/EMAIL-PROVIDERS.md) 和 [隐私与本地数据说明](https://github.com/QinIndexCode/nami-mail/blob/main/docs/PRIVACY.md)。
