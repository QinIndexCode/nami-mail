# Release Notes

此目录保存可审阅、可复用的 GitHub Release 正文。每份版本说明都应面向实际使用者，而不是复制构建日志、CI 输出或内部环境变量。

- [v0.1.0](v0.1.0.md) 是首个公开版本的可直接粘贴说明。
- [v0.1.1 候选说明](v0.1.1-candidate.md) 是后续真实线上升级验收的模板；其中的候选措辞不能在验收完成前改写为“已验证”。

## 使用规则

1. 在 GitHub Release 页面填写版本标题后，复制对应版本说明的正文；不要把私钥、证书、令牌、测试账户或本机路径写入 Release Notes。
2. 先核对 Release 页面恰好有五项指定资源、版本 tag 与安装程序名称一致，再填写签名状态和已知限制。
3. 只有完成 [发布后真实更新验证](../RELEASING.md#发布后真实更新验证) 并保留可复查证据后，才能在下一版说明中写“已完成线上自动更新验收”。
4. 若资源、签名、版本或更新清单发生变化，先撤销或标记该 Release，再重新走发布验证；不要静默替换资源后沿用旧说明。

用户应始终下载 Release 中的 `Nami Mail Setup <version>.exe`。版本化 ZIP、JSON 清单、`latest.yml` 与 `.blockmap` 是发行和自动更新内部资源，不是手动安装包。
