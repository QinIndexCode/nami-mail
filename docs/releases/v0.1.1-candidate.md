## 发布前必须核实的内容

将上面的候选内容粘贴到 GitHub Release 前，逐项完成并保留证据：

- [ ] `v0.1.1` tag 与 `package.json` 版本完全一致，且 Release 页面恰好有五项指定资源；
- [ ] `Nami Mail Setup 0.1.1.exe` 的 Authenticode 状态、发布者和 SmartScreen 说明已按实际结果填写；
- [ ] `nami-mail-update-0.1.1-win-x64.json` 的 Ed25519 签名已由客户端信任根验证，ZIP 的大小与 SHA-512 已复核；
- [ ] 使用真实安装的 `0.1.0`，在公开网络下分别验证更新提示的“更新此版本”“跳过此版本”“稍后提醒”；
- [ ] 完成下载、重启安装、旧版退出、新版启动、本地账户和设置保留，以及 ZIP/临时目录受限清理的验证；
- [ ] 记录网络中断、TLS 失败、404 或缺失资源、篡改清单或 ZIP、签名失败、同步无法安全关闭和重复启动的可操作结果；
- [ ] 所有公开文案只包含已验证事实，不包含测试账户、令牌、证书、私钥、真实邮件或本机路径。
