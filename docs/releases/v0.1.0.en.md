# Nami Mail 0.1.0

[简体中文](v0.1.0.md) | [English](v0.1.0.en.md)

Nami Mail 0.1.0 is the first public release for local-first, multi-account mail use on Windows x64.

## Download and Get Started

Download only `Nami Mail Setup 0.1.0.exe` from this Release. After installation, follow the in-app guidance to add an account. Prefer OAuth for Gmail and Outlook/Microsoft 365. Use the app-specific password or client authorization code required by the provider for QQ, NetEase, iCloud, and similar accounts.

Do not manually extract or run `nami-mail-update-0.1.0-win-x64.zip`, `nami-mail-update-0.1.0-win-x64.json`, `latest.yml`, or `.blockmap`. They are for release validation and in-app automatic updates, not installation.

## What This Release Provides

- A unified inbox, multi-account synchronization, search, unread state, drafts, sending, and attachment downloads.
- Common-provider discovery, manual IMAP/SMTP configuration, and Google and Microsoft OAuth sign-in entry points.
- Selectable mail text plus verification-code detection and one-click copying.
- Windows single-instance behavior, tray behavior, themes, and responsive UI.
- Local SQLite storage with application-layer encryption for credentials, sensitive mail payloads, the outbound queue, and outbound attachments.

See the [email provider setup guide](https://github.com/QinIndexCode/nami-mail/blob/main/docs/EMAIL-PROVIDERS.en.md) for account preparation, authentication differences, and connection troubleshooting. See [privacy and local data](https://github.com/QinIndexCode/nami-mail/blob/main/docs/PRIVACY.en.md) for local-data scope and encryption boundaries.

## Important Signing and SmartScreen Information

The Windows installer in this release is **not Authenticode-signed**. Windows SmartScreen can show an "unknown publisher" or other origin warning. That is not proof that the file is trustworthy, and it is not a reason to bypass system protection.

Before installation, return to the [QinIndexCode/nami-mail Releases page](https://github.com/QinIndexCode/nami-mail/releases) and verify the repository, version tag, installer name, and this note. If the source, version, or signing state is unclear, stop and report the problem through the [support guide](https://github.com/QinIndexCode/nami-mail/blob/main/SUPPORT.en.md).

## Automatic Update Status

The versioned update manifest, `nami-mail-update-0.1.0-win-x64.json`, is signed with Ed25519. The app uses it to bind the update ZIP version, name, size, and SHA-512. The manifest is update trust material, not a file to download or run manually.

`0.1.0` is the first public release. The real online verification of discovering, downloading, validating, restarting, and upgrading an installed `0.1.0` to `0.1.1` through public GitHub Releases is **still pending when v0.1.1 is released**. This note therefore does not describe local builds, static checks, or the presence of Release assets as verified online automatic updates.

The app offers **Update this version**, **Skip this version**, and **Remind me later**. Until the real upgrade verification above is complete, treat those as implemented product paths rather than a completed online-availability conclusion.

## Known Limitations

- The current public desktop release scope is Windows x64.
- Google and Microsoft OAuth depend on correct public-client configuration from the deployer, and organization administrators can restrict IMAP or OAuth.
- Local encryption is not full database encryption and cannot defend against an unlocked app, an administrator, or a malicious process able to call DPAPI as the same Windows user.
- Provider protocols, account plans, and administrator policy can affect connectivity. Official provider help linked by the app and provider documentation take precedence.
