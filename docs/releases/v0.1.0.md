# Nami Mail 0.1.0

Nami Mail 0.1.0 是首个公开版本，面向 Windows x64 的本地优先多账户邮件使用场景。

## 下载与开始使用

请只下载本 Release 中的 `Nami Mail Setup 0.1.0.exe`。安装后，按应用内引导添加邮箱；Gmail、Outlook/Microsoft 365 等账户优先使用 OAuth，QQ、网易、iCloud 等账户请使用服务商要求的应用专用密码或客户端授权码。

不要手动解压或运行 `nami-mail-update-0.1.0-win-x64.zip`、`nami-mail-update-0.1.0-win-x64.json`、`latest.yml` 或 `.blockmap`。它们用于发行校验和应用内自动更新，不是安装包。

## 本次提供的能力

- 统一收件箱、多账户同步、搜索、未读状态、草稿、发送和附件下载；
- 常见邮箱服务商识别、IMAP/SMTP 手动配置，以及 Google 和 Microsoft 的 OAuth 登录入口；
- 可复制的邮件文本和验证码识别与一键复制；
- Windows 单实例、托盘行为、主题与响应式界面；
- 本地 SQLite 存储，以及对凭据、敏感邮件载荷、发件队列和出站附件的应用层加密。

详细的邮箱准备、认证差异和连接排障见[邮箱接入指南](https://github.com/QinIndexCode/nami-mail/blob/main/docs/EMAIL-PROVIDERS.md)。本地数据范围与加密边界见[隐私与本地数据说明](https://github.com/QinIndexCode/nami-mail/blob/main/docs/PRIVACY.md)。

## 重要的签名与 SmartScreen 说明

此版本的 Windows 安装程序**没有 Authenticode 签名**。Windows SmartScreen 可能显示“未知发布者”或其他来源警告；这不是文件可信的证明，也不应因为看到该提示就绕过系统保护。

安装前请回到 [QinIndexCode/nami-mail 的 Releases 页面](https://github.com/QinIndexCode/nami-mail/releases)，核对仓库、版本 tag、安装程序名称和本说明。来源、版本或签名状态不明确时，请停止安装并通过[支持指南](https://github.com/QinIndexCode/nami-mail/blob/main/SUPPORT.md)报告。

## 自动更新状态

本 Release 的版本化 JSON 更新清单 `nami-mail-update-0.1.0-win-x64.json` 使用 Ed25519 签名，应用会用它绑定 ZIP 的版本、名称、大小和 SHA-512。该清单是自动更新的信任材料，不是手动下载或执行的文件。

`0.1.0` 是首个公开版本。使用已安装的 `0.1.0` 从公开 GitHub Release 发现、下载、校验、重启并升级到 `0.1.1` 的真实线上验收，**仍待在 v0.1.1 发布时完成**。因此，本说明不把本地构建、静态检查或 Release 资源存在本身表述为“线上自动更新已验收”。

应用提供“更新此版本”“跳过此版本”和“稍后提醒”选项；在上述真实升级验收完成前，请将其视为已实现的功能路径，而不是已完成的线上可用性结论。

## 已知限制

- 当前公开桌面发行范围为 Windows x64；
- Google 和 Microsoft OAuth 依赖部署者提供正确的公共客户端配置，组织管理员也可能限制 IMAP 或 OAuth；
- 本地加密不是整库加密，不能防御已解锁应用、管理员或同一 Windows 用户下可调用 DPAPI 的恶意进程；
- 邮箱服务商的协议、账户套餐和管理员策略可能影响连接能力，应用内官方帮助入口和服务商说明优先。

更多安装、更新、卸载和 SmartScreen 处理说明见[Windows 安装与更新指南](https://github.com/QinIndexCode/nami-mail/blob/main/docs/INSTALLING.md)。问题反馈与支持范围见[支持指南](https://github.com/QinIndexCode/nami-mail/blob/main/SUPPORT.md)。
